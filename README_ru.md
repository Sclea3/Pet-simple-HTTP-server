# Пет-проект: Простой HTTP сервер (возможно gRPC в будущем) с Docker

## Описание

Необходимо реализовать простой HTTP сервер на Go для управления пользователями. Основная логика приложения включает создание пользователя и управление его задачами, такими как ввод реферального кода, подписка на Telegram или Twitter, и получение награды в виде баллов. Награды можно настроить для каждой задачи, а также добавить другие задачи на ваше усмотрение.

## Требуемая функциональность

### Основные функции

- **Middleware для авторизации** с использованием Access токенов для всех эндпоинтов (например, JWT)
- Реализация **HTTP API**:
    - `GET /users/{id}/status` – получение всей доступной информации о пользователе
    - `GET /users/leaderboard` – получение топа пользователей с наибольшим балансом
    - `POST /users/{id}/task/complete` – выполнение задания
    - `POST /users/{id}/referrer` – ввод реферального кода (может быть id другого пользователя)
- **Хранение данных** в PostgreSQL для всех данных о пользователях (с использованием миграций через `golang-migrate`)
- **Docker Compose** для сборки и запуска проекта

### Дополнительные требования

- Обработка **ошибок** (например, неверные данные, несуществующие пользователи и т.д.)
- Соблюдение принципов **SOLID** для чистого и поддерживаемого кода

## Структура проекта

Проект должен придерживаться стандартной структуры для Go проектов:

### `/cmd`
- Основное приложение (`main.go`)

### `/internal`
- Внутренний код и библиотеки, которые не используются вне приложения

### `/pkg`
- Не используется в этом проекте, так как код не предполагается для внешнего использования

### `/api`
- Спецификации OpenAPI/Swagger, JSON схема, определения протоколов

### `/configs`
- Конфигурационные файлы и настройки по умолчанию

### `/build`
- Для CI/CD (`/build/ci`), Docker (`/build/package`)

### `/deploy`
- Подготовка для Kubernetes и docker-compose

### `/test`
- Юнит-тесты для приложения
